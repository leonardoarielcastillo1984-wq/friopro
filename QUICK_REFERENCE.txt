}

Response:
{
  "user": { "id": "...", "email": "..." },
  "activeTenant": { "id": "...", "name": "...", "slug": "..." },
  "tenantRole": "TENANT_ADMIN",
  "csrfToken": "...",
  "accessToken": "..." (in cookie)

Status: 200 OK


WHAT'S DIFFERENT
================

BEFORE:
- 2FA required on every login
- 3 screens shown
- 2FA session token returned
- Requires authentication code

AFTER:
- 2FA optional (not enforced)
- 2 screens shown
- Access token returned directly
- Just email + password needed

✓ All other endpoints unchanged
✓ 2FA infrastructure still available

TROUBLESHOOTING
===============

Problem: Getting "requires2FA" in response
Fix: Make sure you're using the updated backend code

Problem: 2FA form still appearing
Fix: Clear browser cache or reload test-login.html

Problem: Login fails with invalid credentials
Check: Email and password are correct (case-sensitive)

Problem: Want to restore 2FA
Solution: Restore from git history or see CODE_CHANGES_SUMMARY.txt

SECURITY
========

Password Requirements:
- Minimum 8 characters
- Case-sensitive
- Hashed with Argon2

Token Security:
- HTTP-only cookies
- Secure flag (HTTPS in production)
- SameSite=Lax
- JWT with expiration

Rate Limiting:
- 10 requests per minute (strict: 5/minute)
- Per IP address
