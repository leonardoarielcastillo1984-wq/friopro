PROBLEMA #1 - PUERTO INCORRECTO
  • Frontend intenta conectar a API en puerto 3001 (INCORRECTO)
  • API en realidad corre en puerto 3002
  • Resultado: "fetch failed" + ECONNREFUSED
  FIX: ✅ /apps/web/.env → NEXT_PUBLIC_API_URL=http://localhost:3002

PROBLEMA #2 - npm install FALLANDO
  • bullmq@5.66.5 postinstall hook ejecuta "husky install"
  • husky no estaba instalado globalmente
  • Resultado: API NUNCA se levanta
  FIX: ✅ npm install -g husky (ejecutado en script)

PROBLEMA #3 - DATABASE URLs con Docker hostnames
  • API .env usaba: postgres:5432 y redis:6379 (Docker network)
  • Para desarrollo local deberían ser: localhost:5432 y localhost:6379
  FIX: ✅ /apps/api/.env actualizadas a localhost

PROBLEMA #4 - .env SUCIO Y DUPLICADO
  • /apps/api/.env tenía 3x líneas duplicadas (HOST, CORS_ORIGIN)
  FIX: ✅ Archivo limpiado

PROBLEMA #5 - USUARIO NO EXISTE EN BD
  • admin@sgi360.com nunca se creó en la base de datos
  • Login siempre fallaba con "Invalid credentials"
  FIX: ✅ Script ejecuta npm run seed:complete automáticamente
