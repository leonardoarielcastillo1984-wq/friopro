# SGI 360 Database Implementation

## Overview

This implementation provides a complete PostgreSQL database setup for SGI 360, including Docker orchestration, Prisma ORM, migrations, and seeding scripts for the 2FA system.

## What's Included

### 1. Docker Setup (`docker-compose.yml`)

Complete containerized environment with:
- **PostgreSQL 16** - Production-ready database
- **Redis 7** - Caching and session storage
- **Node API** - Fastify backend server
- **Next.js Frontend** - React-based web interface

**Features**:
- Health checks for all services
- Volume persistence for data
- Network isolation
- Development mode with hot-reload
- Proper startup order and dependencies

### 2. Environment Configuration

- **`.env.local`** - Local development variables
- **`.env.example`** - Template with all variables
- **`.env.staging`** - Staging environment (from existing)
- **`docker-compose.prod.yml`** - Production configuration

### 3. Database Initialization

**`infra/postgres-init/01-init.sql`**:
- Creates application user
- Enables required PostgreSQL extensions (uuid-ossp, pg_trgm, unaccent)
- Sets up permissions and privileges
- Creates helper functions for audit trails

### 4. Prisma Schema (`apps/api/prisma/schema.prisma`)

Already configured with 2FA models:
- **TwoFactorAuth** - TOTP secret and configuration
- **TwoFactorRecoveryCode** - Backup codes (hashed)
- **TwoFactorSession** - Temporary verification sessions
- All supporting tables (User, Tenant, Audit, etc.)

### 5. Database Migrations

Located in `apps/api/prisma/migrations/`:
- 0001-0010: Core application migrations
- **0007_two_factor_auth**: 2FA tables and indexes
- All migrations are version-controlled and reversible

### 6. Seeding Scripts

#### Main Seed Scripts
- **`apps/api/src/scripts/seedPlans.ts`** - Subscription plans
- **`apps/api/src/scripts/seedUsers.ts`** - Test users
- **`apps/api/src/scripts/seedDemoData.ts`** - Sample application data

#### 2FA Seed Script

  - Creates test users with 2FA enabled
  - Generates 10 recovery codes per user
  - Creates test TOTP secret (JBSWY3DPEBLW64TMMQ======)
  - Creates sample 2FA sessions

### 7. Setup and Management Scripts

- **`scripts/setup-local-dev.sh`** - One-command setup
- **`scripts/run-migrations.sh`** - Migration management
- **`scripts/seed-2fa.sh`** - 2FA data seeding

### 8. Documentation

- **`DATABASE_SCHEMA.md`** - Complete schema documentation
- **`POSTGRESQL_SETUP_GUIDE.md`** - Setup and troubleshooting
- **`DATABASE_README.md`** - This file

## Quick Start

### Option 1: Automated Setup (Recommended)

```bash
./scripts/setup-local-dev.sh
```

This runs everything:
1. Copies environment files
2. Installs dependencies
3. Starts Docker services
4. Runs migrations
5. Seeds test data
6. Starts API and Frontend

### Option 2: Manual Setup

```bash
# 1. Start databases
docker-compose up -d postgres redis

# 2. Install dependencies
pnpm install
cd apps/api
npm install

# 3. Generate Prisma client
npm run prisma:generate

# 4. Run migrations
npm run prisma:migrate

# 5. Seed data
npm run seed:all
npm run --env=local seed:2fa

# 6. Start application
cd ../..
docker-compose up api web
```

## Database Tables

### 2FA Tables

#### `TwoFactorAuth`
Stores TOTP secrets and 2FA status.

| Column | Type | Key | Notes |
|--------|------|-----|-------|
| id | UUID | PK | Generated by database |
| userId | UUID | UNIQUE FK | One per user |
| secret | TEXT | - | Base32 TOTP secret |
| isEnabled | BOOLEAN | - | True if active |
| isConfirmed | BOOLEAN | - | True after setup |
| createdAt | TIMESTAMPTZ | - | Creation time |
| enabledAt | TIMESTAMPTZ | - | When activated |

#### `TwoFactorRecoveryCode`
Backup codes (one-time use, hashed).

| Column | Type | Key | Notes |
|--------|------|-----|-------|
| id | UUID | PK | Generated |
| twoFactorAuthId | UUID | FK | Links to TwoFactorAuth |
| code | VARCHAR | UNIQUE | SHA256 hash |
| used | BOOLEAN | - | One-time use |
| usedAt | TIMESTAMPTZ | - | When consumed |

#### `TwoFactorSession`
Temporary sessions during verification.

| Column | Type | Key | Notes |
|--------|------|-----|-------|
| id | UUID | PK | Generated |
| userId | UUID | FK | User verifying |
| sessionToken | VARCHAR | UNIQUE | Unique per session |
| verified | BOOLEAN | - | TOTP verified? |
| expiresAt | TIMESTAMPTZ | - | 10 min default |

## Key Features

### 2FA System

**Setup Flow:**
1. User requests 2FA setup
2. Server generates TOTP secret
3. Server returns QR code and manual key
4. User scans QR code in authenticator app
5. User enters code to confirm
6. Server generates recovery codes
7. User saves recovery codes securely

**Recovery Flow:**
1. User loses 2FA device
2. User uses recovery code instead of TOTP
3. Recovery code marked as used
4. User can request new recovery codes

**Session Management:**
1. Create temporary 2FA session (10 min)
2. User submits TOTP or recovery code
3. Verify and mark session as verified
4. Issue JWT token for authenticated session
5. Clean up expired sessions

### Database Features

- **UUID Primary Keys** - No sequential IDs exposed
- **Soft Deletes** - Data retention and audit trails
- **Timestamps** - Full audit with created/updated times
- **Indexes** - Optimized for common queries
- **Constraints** - Data integrity enforcement
- **Relationships** - Proper foreign keys with cascades

### Security

- **Password Hashing** - Argon2 via Prisma
- **Code Hashing** - SHA256 for recovery codes
- **Session Tokens** - 32-byte random tokens
- **Field Encryption** - Ready for TDE (Transparent Data Encryption)
- **RLS Policies** - Row-level security in PostgreSQL

## Troubleshooting

### Services Won't Start

```bash
# Check logs
docker-compose logs postgres
docker-compose logs redis

# Restart everything
docker-compose down -v
docker-compose up postgres redis
```

### Migration Errors

```bash
# Check status
cd apps/api
npx prisma migrate status

# Reset to clean state
npm run prisma:reset

# Try again
npm run prisma:migrate
```

### Port Conflicts

```bash
# See what's using ports
lsof -i :5432  # PostgreSQL
lsof -i :6379  # Redis
lsof -i :3001  # API
lsof -i :3000  # Frontend

# Use different ports
# Edit docker-compose.yml and restart
```

### Database Connection Errors

```bash
# Test connection
psql postgresql://sgi:sgidev123@localhost:5432/sgi_dev -c "SELECT 1"

# From Docker
docker-compose exec postgres psql -U sgi -d sgi_dev -c "SELECT 1"
```

### Seed Data Missing

```bash
# Check what was seeded
cd apps/api
npm run seed:all --verbose

# Reseed specific
npm run seed:all
npm run seed:demo
npm run --env=local seed:2fa
```

## Best Practices

1. **Always backup before migrations** - `pg_dump` to backup
2. **Test migrations locally first** - Don't run on production directly
3. **Use .env.local for development** - Never commit to git
4. **Rotate secrets regularly** - JWT_SECRET, REDIS_PASSWORD
5. **Monitor database logs** - Enable slow query log
6. **Archive old data** - Clean up expired sessions
7. **Document changes** - Keep migration notes updated
8. **Version control migrations** - All changes tracked

## Performance Tips

1. **Connection Pooling** - Use PgBouncer for production
2. **Index on Frequently Filtered Columns** - 2FA enabled, session expiry
3. **Analyze Table Statistics** - Help query planner
4. **Use EXPLAIN ANALYZE** - Debug slow queries
5. **Archive Historical Data** - Keep tables lean
6. **Vacuum Regularly** - Maintain database health
